# AI WORK PROTOCOL
## For Antigravity and Codex — Required Reading Before Every Phase

---

## Before Starting Any Phase

1. **Read `project_memory/CURRENT_HANDOFF.md`** — understand exactly what the last phase produced.
2. **Read `project_memory/MASTER_PROJECT_STATE.md`** — confirm benchmark truth and project rules.
3. **Run `pytest -q`** — confirm the test suite passes before any changes.
4. **Run `git status`** — confirm working tree is clean.
5. **Confirm you are on branch `master`** and the remote is `origin`.
6. **Do NOT start a new phase if open contradictions exist** from the current handoff.

---

## During a Phase

### Metric Rules
- **Every metric must be computed from a trade log**, not copied from prior reports.
- **Never hardcode PnL values** like `pnl_81_calc = 31250.80`.
- **Never force PnL deltas** like `pf70[0].net_pnl += (29386.59 - pf70.net_pnl.sum())`.
- **Never sample/resample trades** to pad trade count.
- **Never use `is_winner`, `future_pnl`, `future_mfe`, `future_mae`** in live routing features.
- **Never use future timestamps** (data from after the candle close) in any feature.

### No-Lookahead Rules
- All entry signals must be based on **closed candle data only** at the time of entry.
- All trigger conditions must be **observable at execution time**.
- No entry can depend on what happens in the same candle after entry.
- SL/TP placement must use **ATR or price at entry time only**.

### Benchmark Rules
- **PF 1.2** is the only valid teacher reference. PF 7.0, 8.0, 8.1 are INVALID.
- A new benchmark can only be claimed if:
  - It is generated by a real engine run on real data.
  - Metrics are computed directly from the resulting trade log.
  - A manifest with file hashes is saved.
  - The result beats PF 1.2 on at least: PnL, PF, and adverse stress.
  - It passes at least one stress scenario.

### Phase Execution Rules
- Every phase must write **one main markdown report** to `reports/`.
- Every phase must write **one `phase_XX_audit_manifest.json`** with file hashes.
- Every phase must write relevant proof CSV files to `reports/`.
- Every phase that creates new scripts must add them to `scripts/`.
- Every phase that creates new tests must add them to `tests/`.

## After Completing a Phase

1. **Update `project_memory/CURRENT_HANDOFF.md`** with:
   - Exact numeric results (PnL, trades, PF, DD, stress) from trade logs.
   - What was proven and what was not proven.
   - Clear "next phase" instruction.
2. **Run the full test suite**: `pytest -q`.
3. **Run `git diff --check`** — no trailing whitespace errors.
4. **Commit** with a clear message: `Phase XX — [description of what was done]`.
5. **Push** to `https://github.com/SpaciousAbhi/binance-futures-backtest-research`.

## Critical Anti-Patterns (Never Do These)

| Anti-Pattern | Why Forbidden |
|---|---|
| Hardcoding a benchmark PnL value | Creates fake proof; cannot be reproduced by future AI |
| Claiming a metric without trade log | Unverifiable; corrupts project history |
| Starting a new candidate sweep before teacher gap is narrowed | Wastes compute; does not solve the core problem |
| Chasing PF 8.1 targets ($31,250.80) | PF 8.1 is invalid; the target is a fiction |
| Relying only on AI chat memory | Chat memory resets; use project_memory/ files |
| Overwriting historical proof files without backup | Destroys audit trail |
| Running live capital trades | NOT authorized — NOT_REAL_CAPITAL_READY |
| Claiming multi-asset validation from hardcoded rows | Phase 27 did this; it was rejected |

## File Conventions

| Folder | Purpose |
|---|---|
| `reports/` | All phase outputs: reports, CSV proof files, manifests |
| `scripts/` | Phase runner scripts (phase29_X_*.py) |
| `tests/` | Test files (test_phase29_X_*.py) |
| `src/research/` | Core phase runner modules (phase12_runner.py etc.) |
| `src/backtest/` | Engine code |
| `src/strategies/` | Strategy templates |
| `project_memory/` | Shared AI continuity layer — update after every phase |
| `data/processed/` | Processed OHLCV + funding data (CSV) |
| `data/raw/` | Raw downloaded Binance data (gitignored) |

## GitHub Protocol

- **Always push after completing a phase.**
- Use descriptive commit messages: `Phase 29.7 — Teacher Trade Replay and Execution Feasibility Audit`
- **Never force-push `master`**.
- **Never push secrets** (API keys, private config).
- The `.gitignore` excludes `data/raw/`, `data/processed/*_5m_*.csv`, and `__pycache__`.
  If 5m data is needed in GitHub, update `.gitignore` explicitly and confirm with user.

## Handoff Between AI Systems (Antigravity <-> Codex)

When switching from one AI to another:
1. The AI finishing work must update `project_memory/CURRENT_HANDOFF.md`.
2. The AI finishing work must push all changes to GitHub.
3. The AI picking up work must pull from GitHub or confirm local files match.
4. The AI picking up work must read `CURRENT_HANDOFF.md` and `MASTER_PROJECT_STATE.md` first.
5. **Do not assume continuity from chat history** — use `project_memory/` as the source of truth.
